# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in FerroML, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please use [GitHub Security Advisories](https://github.com/robertlupo1997/ferroml/security/advisories/new) to report the vulnerability privately.

## Scope

This policy applies to:

- `ferroml-core` (Rust crate)
- `ferroml` (Python package via PyPI)
- Build and CI/CD infrastructure

## Response

We will acknowledge receipt within 48 hours and aim to provide a fix or mitigation plan within 7 days for confirmed vulnerabilities.

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.3.x   | Yes       |
| < 0.3   | No        |

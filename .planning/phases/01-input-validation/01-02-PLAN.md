       - All preprocessors/transformers (StandardScaler, MinMaxScaler, RobustScaler, etc.) -- ensure transform() validates input.
       - All models with predict_proba() -- ensure NaN/Inf check on input.

    **Part C: Fix silent hyperparameter clamping (VALID-08)**

    8. In `ferroml-core/src/models/svm.rs`, find all `with_*` builder methods that use `.max()` or `.min()` for silent clamping (research identified `with_c`, `with_tol`). Remove the clamping and instead add validation at the START of fit():
       ```rust
       // At top of SVC::fit():
       if self.c <= 0.0 {
           return Err(FerroError::invalid_input(
               &format!("Parameter C must be positive, got {}", self.c)
           ));
       }
       if self.tol <= 0.0 {
           return Err(FerroError::invalid_input(
               &format!("Parameter tol must be positive, got {}", self.tol)
           ));
       }
       ```
       Do the same for SVR, LinearSVC, LinearSVR.

    9. Audit ALL models in `ferroml-core/src/models/` for similar silent clamping patterns (grep for `.max(` and `.min(` in `with_` methods). Convert each to fit-time validation. Common params to check: learning_rate, max_depth, n_estimators, alpha, l1_ratio, n_neighbors, bandwidth.

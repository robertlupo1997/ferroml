  <action>
    1. Create `ferroml-core/src/validation.rs` with these public functions:
       - `validate_unsupervised_input(x: &Array2<f64>) -> Result<()>` -- checks empty (nrows==0), NaN/Inf via `!is_finite()`. Error messages should include count and first position of offending values: "X contains N NaN/Inf values (first at row R, column C)".
       - `validate_transform_input(x: &Array2<f64>, expected_features: usize) -> Result<()>` -- checks feature count mismatch, empty, NaN/Inf. Same detailed error messages.
       - Re-export `check_is_fitted` from here as well (or keep it in models/mod.rs -- just ensure it is accessible). Do NOT move `validate_fit_input` -- it stays in `models/mod.rs` to avoid breaking 30+ existing imports. The new module complements, not replaces.

    2. Add `pub mod validation;` to `ferroml-core/src/lib.rs`.

    3. Refactor all 5 clustering models (KMeans, DBSCAN, GMM, HDBSCAN, Agglomerative):
       - Replace inline NaN/empty checks in `fit()` with `crate::validation::validate_unsupervised_input(x)?;` at the top of fit.
       - Replace inline checks in `predict()` with `crate::validation::validate_transform_input(x, self.n_features)?;` (or equivalent stored feature count).
       - Keep model-specific validation (e.g., KMeans checking n_clusters <= n_samples) AFTER the shared call.

    4. Add validation to all 5 decomposition models (PCA, t-SNE, LDA, TruncatedSVD, FactorAnalysis):
       - Add `crate::validation::validate_unsupervised_input(x)?;` as the first line of each `fit()` method.
       - Add `crate::validation::validate_transform_input(x, self.n_features_in)?;` at the top of each `transform()` method (where applicable).
       - For models that have `fit_transform`, validate in the fit portion.

    5. Add hyperparameter validation at the START of `fit()` for clustering and decomposition models (VALID-08). Do NOT change builder method signatures. Examples:
       - KMeans: `if self.n_clusters == 0 { return Err(FerroError::invalid_input("Parameter n_clusters must be >= 1, got 0")); }`
       - PCA: `if self.n_components == 0 { return Err(FerroError::invalid_input("Parameter n_components must be >= 1, got 0")); }`
       - GMM: `if self.n_components == 0 { ... }`
       - For tolerance params: `if self.tol <= 0.0 { ... }`
       - For any silently clamped params: remove the clamping, add validation.

    6. Run `cargo fmt --all` before testing.

    IMPORTANT: Do NOT add new test binary files. Tests go in existing `ferroml-core/tests/edge_cases.rs`.
    IMPORTANT: Do NOT move or rename existing `validate_fit_input` -- it has 30+ callers.
    IMPORTANT: Keep Rust-side validation even though Python-side will be added later (defense in depth).

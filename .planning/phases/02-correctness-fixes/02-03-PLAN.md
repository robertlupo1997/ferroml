**A. Output validation utility (CORR-05):**
1. Add to `models/mod.rs`:
```rust
/// Check model output for NaN/Inf values. Returns error if any found.
pub fn validate_output(predictions: &Array1<f64>, model_name: &str) -> Result<()> {
    if let Some(pos) = predictions.iter().position(|v| !v.is_finite()) {
        return Err(FerroError::numerical(format!(
            "{} produced non-finite output at index {} (value: {}). \
             This indicates a numerical issue in the model.",
            model_name, pos, predictions[pos]
        )));
    }
    Ok(())
}
```
2. Add a 2D variant `validate_output_2d` for `predict_proba` results
3. Add unit tests: all-finite passes, NaN detected, Inf detected, correct position reported
4. Wire into predict() for models most likely to produce NaN: LogisticRegression, GaussianNB, GMM. Add `validate_output(&predictions, "ModelName")?;` before returning predictions. Do NOT wire into every model yet -- start with the highest-risk ones. Other models can adopt it incrementally.

**B. Convergence warning mechanism (CORR-10):**
1. Add to `error.rs`:
```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum ConvergenceStatus {
    Converged { iterations: usize },
    NotConverged { iterations: usize, final_change: f64 },
}
```
2. Modify iterative model fit() methods to:
   - Store `convergence_status: Option<ConvergenceStatus>` field
   - On non-convergence: call `log::warn!("{} did not converge after {} iterations...", ...)` and return the best partial result instead of erroring
   - IMPORTANT: Only change models that currently hard-error on non-convergence. If a model already returns partial results, just add the status field and warning.
3. Update these models (check each for current behavior first):
   - KMeans: should warn and return best centroids
   - GMM: should warn and return best parameters
   - LogisticRegression (if using iterative solver): should warn
   - SVM: keep hard error for "no support vectors" case (legitimate failure), but warn for max_iter reached with partial solution
4. Add getter method `convergence_status(&self) -> Option<&ConvergenceStatus>` to each modified model
5. Add unit tests: model that converges returns Converged status, model with max_iter=1 returns NotConverged with warning

Run: `cargo test -p ferroml-core`

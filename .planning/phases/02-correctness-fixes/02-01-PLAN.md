The research identified these issues:
1. The gradient descent optimizer uses fixed learning rate (0.01) with no line search or adaptive step
2. Temperature is clamped to >= 0.01 but gradient has 1/T^2 factor that amplifies near small T
3. The optimizer may oscillate or converge to the clamp boundary

Fix approach (from research): Replace the hand-rolled gradient descent with either:
- Golden-section search (simple, 1D problem, guaranteed convergence)
- L-BFGS via argmin (already a dependency, used by LogisticRegression)
- Brent's method for 1D optimization

The simplest correct approach: since this is a 1D convex optimization of NLL w.r.t. temperature T, use a bounded scalar optimization method. Golden-section search on [0.01, 10.0] is the most robust.

First, run the failing tests to capture exact error output:


The research identified these issues:
1. The variance combination formula at line ~889 has a suspicious extra division by `new_n`: `old_n * n_samples / (new_n * new_n)` should likely be `old_n * n_samples / new_n`
2. The augmented SVD approach (Ross et al.) for incremental components looks correct

Fix approach:
1. Fix the variance combination formula to use correct Chan-Golub-Welford incremental formula
2. The correct incremental variance update is:
   - `new_var = (old_var * old_n + batch_var * n_samples + correction) / new_n`
   - where `correction = (old_mean - batch_mean)^2 * old_n * n_samples / new_n`
3. Verify explained_variance_ratio_ sums to <= 1.0
4. Cross-check components match batch PCA on same data

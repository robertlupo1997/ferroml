**Step 1: Create ferro_to_pyerr in errors.rs**

Add a new function `ferro_to_pyerr` that maps FerroError variants to semantically correct Python exceptions, per the mapping table in RESEARCH.md:
- InvalidInput, ShapeMismatch, ConfigError -> PyValueError
- NotFitted -> PyRuntimeError (with clear "call fit() first" message)
- NumericalError, ConvergenceFailure, AssumptionViolation -> PyRuntimeError
- NotImplemented, NotImplementedFor -> PyNotImplementedError
- SerializationError, CrossValidation, InferenceError -> PyRuntimeError
- IoError -> PyIOError
- Timeout -> PyTimeoutError
- ResourceExhausted -> PyRuntimeError (MemoryError would require unsafe, use RuntimeError)

The function takes a `ferroml_core::FerroError` (not generic Display) and returns `PyErr`. Keep `to_py_runtime_err` as a deprecated alias for backward compatibility during transition. Keep `not_fitted_err` unchanged.

**Step 2: Replace to_py_runtime_err with ferro_to_pyerr across all binding files**

In every `.rs` file under `ferroml-python/src/` that uses `to_py_runtime_err`, replace with `ferro_to_pyerr` where the error type is `FerroError`. For cases where the error is NOT a FerroError (e.g., serde errors, string errors), keep `to_py_runtime_err` or convert to the appropriate PyErr directly.

Search pattern: `.map_err(to_py_runtime_err)` -> `.map_err(ferro_to_pyerr)` where the Result error type is FerroError.

**Step 3: Add pickle support to missing binding files**

The following binding files have 0 `__getstate__`/`__setstate__` methods: svm.rs, naive_bayes.rs, gaussian_process.rs, anomaly.rs, calibration.rs, multioutput.rs, stats.rs.

For each model struct in these files, add pickle methods using the existing `pickle::getstate` and `pickle::setstate` utilities. Follow the pattern from linear.rs or trees.rs which already have pickle support:

```rust
fn __getstate__(&self, py: Python<'_>) -> PyResult<PyObject> {
    pickle::getstate(&self.inner, py)
}
fn __setstate__(&mut self, state: &Bound<'_, PyBytes>) -> PyResult<()> {
    self.inner = pickle::setstate(state)?;
    Ok(())
}
```

Ensure the inner Rust model type derives `Serialize` and `Deserialize`. If any model's inner type does NOT derive these traits, add the derives in ferroml-core (this should already be the case for all models).

  <action>
Replace the manual element-wise loops in `e_step()` (lines 640-679) with ndarray matrix operations.

**Important: SVD is NOT the bottleneck.** Code review confirms `thin_svd` is called only once in `initialize_from_svd()` (line 528), before the EM loop. The GOAL-VERIFICATION.md was wrong about per-iteration SVD. The actual bottleneck is three manual O(n^3) triple loops in e_step() that run every EM iteration. Replacing these with ndarray `.dot()` eliminates element-wise indexing overhead and enables SIMD/cache-friendly matrix multiply.

1. **beta_inv computation** (lines 640-652): Replace triple loop with diagonal-scaled dot product:
   ```rust
   // L_scaled[k, i] = loadings[k, i] * psi_inv[k]
   let mut l_scaled = loadings.clone();
   for k in 0..loadings.nrows() {
       l_scaled.row_mut(k).mapv_inplace(|v| v * psi_inv[k]);
   }
   let mut beta_inv = l_scaled.t().dot(loadings);
   for i in 0..n_factors { beta_inv[[i, i]] += 1.0; }
   ```

2. **bltp computation** (lines 658-667): Replace triple loop with:
   ```rust
   let bltp = beta.dot(&l_scaled.t());
   ```
   This replaces the O(n_factors * n_features * n_factors) manual loop with a single ndarray matrix multiply.

3. **exp_f computation** (lines 670-679): Replace loop with:
   ```rust
   let exp_f = x_centered.dot(&bltp.t());
   ```
   This replaces the O(n_samples * n_factors * n_features) manual loop with a single ndarray matrix multiply. This is the largest hot loop (5000 samples * 50 features * n_factors per iteration).

Also check the `m_step()` for similar manual loop patterns that could use ndarray ops. The m_step already uses .dot() so it should be fine.

After changes, run `cargo test -p ferroml-core -- factor_analysis` to verify correctness is preserved. Run `cargo bench -p ferroml-core --bench benchmarks -- FactorAnalysis` to check for improvement.

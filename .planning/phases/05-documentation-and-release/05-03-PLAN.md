Must include these 3 subsections (per user decision):

1. **RandomForest Parallel Non-Determinism**: RandomForest and ExtraTrees use Rayon for parallel tree construction. Due to work-stealing scheduling, results may vary slightly between runs even with the same random_state. For reproducible results, set n_jobs=1.

2. **Sparse Algorithm Support**: 12 models support sparse input (CSR format) via fit_sparse()/predict_sparse(). For unsupported models, sparse matrices are converted to dense automatically. Very large sparse datasets (>100K features) may cause memory issues during conversion.
